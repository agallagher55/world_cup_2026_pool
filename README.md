# 2026 FIFA World Cup Pool

A pick-em pool for the 2026 FIFA World Cup. Each participant selects 12 teams
(2 per tier) before the tournament starts. Points accumulate as teams win
matches and earn group-stage bonuses.

## How Picks Work

### Tier System

All 48 World Cup teams are pre-sorted into 6 tiers by expected strength.
Each participant picks **2 teams per tier = 12 picks total**.

| Tier | Label | Description |
|------|-------|-------------|
| Tier 1 | Elite Favorites | Top title contenders |
| Tier 2 | Contenders | Strong squads with title potential |
| Tier 3 | Dark Horses | Capable of deep runs |
| Tier 4 | Long Shots | Competitive but unlikely to go far |
| Tier 5 | Underdogs | Will face tough groups |
| Tier 6 | Major Underdogs | Longest of long shots |

### Group Compositions (2026)

## Scoring Rules

### Group Stage Match Points

| Result | Points |
|--------|--------|
| Win (regulation) | +3 |
| Draw | +1 |
| Loss | 0 |

### Knockout Stage Match Points

| Result | Points |
|--------|--------|
| Win (regulation) | +3 |
| Win (AET or penalties) | +2 |
| Loss (AET or penalties) | +1 |
| Loss (regulation) | 0 |

### Group Stage Bonuses

Applied after all 6 group games in a group are complete.
Ties are shared — all tied teams receive the full bonus.

| Bonus | Points |
|-------|--------|
| Win the group (most wins) | +3 |
| Most goals scored in group | +3 |
| Most goals scored in entire tournament | +3 additional |
| Fewest goals conceded in group | +3 |
| Fewest goals conceded in entire tournament | +3 additional |

A team can earn a maximum of **+15 bonus points** if they win their group,
lead the tournament in goals scored, and concede the fewest goals overall.

## Scripts

### `extract_picks.py`

Reads every `.xlsx` in `submissions/` and writes (or refreshes) the **Picks** tab.

```bash
python extract_picks.py
```

- Parses each submission's `User Submission Template` sheet
- Validates that each file has exactly 12 picks (2 per tier); prints a warning for invalid submissions
- Colour-codes each tier's columns in the output tab
- Run this whenever a new submission is added

---

### `extract_scores.py`

Pulls live match results from [football-data.org](https://www.football-data.org)
and updates the workbook.

```bash
# Live data (requires API key)
export FOOTBALL_DATA_API_KEY="your_key_here"
python extract_scores.py

# Pass key directly
python extract_scores.py --api-key YOUR_KEY

# Use bundled mock data (no key needed — useful for testing)
python extract_scores.py --mock
```

**What it updates:**
- Refreshes the **Match Results** tab with all WC2026 fixtures and scores
- Refreshes the **Team Stats** tab (goals, wins, group standings)
- Writes each participant's running TOTAL PTS into the **Picks** tab

**Getting an API key:**
1. Register free at [football-data.org/client/register](https://www.football-data.org/client/register)
2. The free tier covers all WC match data

---

### `statistics.py`

Reads picks directly from `submissions/` and writes (or refreshes) the **Statistics** tab.

```bash
python statistics.py
```

**Three sections:**

1. **Team Pick Counts** — every team grouped by tier, sorted by popularity, showing
   pick count, percentage of pool, and which participants chose each team

2. **Pool Analysis** — for each participant:
   - Popularity score (sum of pick-counts for their teams — higher = more mainstream)
   - Uniqueness score (0–100; 100 = every pick is exclusive to them)
   - Consensus overlap (how many of their picks match the most-popular lineup)
   - Contrarian overlap (how many picks are in the least-popular set)
   - WC group diversity (how many different groups their 12 picks span)
   - Group concentration (most picks from a single group — concentration risk)

   Plus the **Consensus Lineup**: for each tier, the most-commonly picked team(s).

3. **Awards & Badges** — fun per-participant highlights:

   | Badge | Awarded to |
   |-------|-----------|
   | ⭐ Crowd Favourite | Highest popularity score — most mainstream picks |
   | 🎯 Dark Horse | Highest uniqueness score — most contrarian picks |
   | 🐺 Lone Wolf | Most exclusive picks (teams nobody else picked) |
   | 🌍 Globetrotter | Picks span the most different WC groups |
   | 🎲 Group Gambler | Most picks from the same single WC group |
   | 🤝 Twins | The pair with the most picks in common (Jaccard similarity) |

---

### `update_scoring.py`

Creates/refreshes the **Scores** and **Scoring** tabs.

```bash
python update_scoring.py           # rebuild both tabs
python update_scoring.py --scores  # rebuild Scores tab only
python update_scoring.py --scoring # rebuild Scoring tab only
```

**Scores tab:**
- Pre-populated with all 72 group stage matchups (6 games × 12 groups)
- Colour-coded by group for easy navigation
- 32 knockout placeholder rows (Round of 32 → Final)
- Yellow score cells indicate where to enter results manually

**Scoring tab:**
- Leaderboard sorted by total points (current leader highlighted in gold)
- Breakdown columns showing each participant's 12 picks, their tier colour,
  and current points earned per pick
- Recomputes from scratch on every run — re-run after entering scores in the Scores tab
- Applies full scoring rules including group bonuses (only triggered once all
  6 games in a group are complete)

## Typical Workflow

### Before the Tournament

1. Each participant fills in `User Submission Template` in the master workbook
   and saves their file to `submissions/`
2. Run `python extract_picks.py` to populate the **Picks** tab
3. Run `python statistics.py` to generate the **Statistics** tab
4. Run `python update_scoring.py` to set up the **Scores** tab skeleton

### During the Tournament

After each match day:

1. Open `WC2026_Pool.xlsx` → **Scores** tab
2. Enter the final scores in the yellow cells for finished games
   (set Duration to `REGULAR`, `EXTRA_TIME`, or `PENALTY_SHOOTOUT` for knockouts)
3. Run `python update_scoring.py --scoring` to refresh the leaderboard

For automated live scoring (instead of manual entry):

```bash
python extract_scores.py  # pulls from football-data.org API
```
